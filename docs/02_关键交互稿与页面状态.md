## 2. 系统设置页面

### 2.1 页面目标

用户配置系统参数，如处理模式、性能档位、GPU 加速等。

### 2.2 页面布局

```
┌─────────────────────────────────────────────────────────────┐
│ 系统设置                                                     │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│ 处理模式：                                                   │
│ ○ 标准模式（推荐）                                           │
│ ○ 轻量模式（快速，质量可能下降）                             │
│                                                               │
│ 性能档位：                                                   │
│ ○ M1 优化（推荐）                                            │
│ ○ CPU 仅模式                                                 │
│ ○ GPU 加速（如果可用）                                       │
│                                                               │
│ GPU 加速：                                                   │
│ ☑ 启用 GPU 加速（如果可用）                                  │
│                                                               │
│ 输出格式：                                                   │
│ ○ MP4 16:9（推荐）                                           │
│ ○ MP4 9:16                                                   │
│                                                               │
│ [保存] [取消]                                                │
│                                                               │
└─────────────────────────────────────────────────────────────┘
```

### 2.3 显示字段

| 字段 | 类型 | 说明 |
|------|------|------|
| 处理模式 | 单选 | 标准模式或轻量模式 |
| 性能档位 | 单选 | M1 优化、CPU 仅模式、GPU 加速 |
| GPU 加速 | 复选 | 是否启用 GPU 加速 |
| 输出格式 | 单选 | MP4 16:9 或 MP4 9:16 |

### 2.4 用户操作

- **保存**：保存设置
- **取消**：取消修改，返回项目列表

### 2.5 说明

- **处理模式**：
  - 标准模式：使用完整的模型和分析，质量最好，耗时较长
  - 轻量模式：使用简化的模型，处理速度快，质量可能下降

- **性能档位**：
  - M1 优化：针对 MacBook Air M1 优化
  - CPU 仅模式：仅使用 CPU，不使用 GPU
  - GPU 加速：使用 GPU 加速（如果可用）

- **GPU 加速**：
  - 如果系统检测到可用的 GPU，用户可以启用 GPU 加速以提高处理速度

- **输出格式**：
  - MP4 16:9：横屏格式（推荐）
  - MP4 9:16：竖屏格式

| 用户标记 | 复选框 | 是否标记为"必须保留" |
| 低置信提示 | 警告 | 如果故事解析置信度低，显示警告 |

### 6.4 用户操作

#### 操作 1：合并段落
- **触发**：点击"合并到下一段"
- **行为**：当前段落与下一段落合并，生成新的摘要
- **反馈**：段落列表更新，显示合并后的结果

#### 操作 2：删除段落
- **触发**：点击"删除此段"
- **行为**：删除当前段落
- **反馈**：段落列表更新；如果删除后段落数过少，显示警告

#### 操作 3：调整段落顺序
- **触发**：拖拽段落卡片
- **行为**：段落顺序改变
- **反馈**：实时更新段落顺序

#### 操作 4：标记"必须保留"
- **触发**：点击"标记为必须保留"复选框
- **行为**：标记当前段落为"必须保留"
- **反馈**：复选框状态改变；该标记会传递到后续的高光确认和成片生成

#### 操作 5：编辑摘要
- **触发**：点击"编辑摘要"
- **行为**：打开文本编辑框，允许用户修改摘要
- **反馈**：摘要更新

#### 操作 6：一键重新生成故事骨架
- **触发**：点击"一键重新生成故事骨架"按钮
- **行为**：恢复到系统推荐的故事骨架版本，丢弃所有用户修改
- **反馈**：段落列表恢复到初始状态；显示确认对话框

### 6.5 状态反馈

| 状态 | 显示 | 说明 |
|------|------|------|
| 正常 | 段落卡片正常显示 | 无问题 |
| 低置信 | 黄色警告标记 | 故事解析置信度低，建议用户审阅 |
| 段落过少 | 红色警告 | 删除后段落数过少，可能影响成片质量 |
| 已修改 | 蓝色指示器 | 表示用户已修改此段落 |

### 5.6 确认与返回

### 6.4 用户操作

#### 操作 1：接受推荐镜头
- **触发**：点击"接受"按钮
- **行为**：接受系统推荐的镜头
- **反馈**：镜头被标记为"已接受"，进入下一段落

#### 操作 2：替换候选镜头
- **触发**：点击备选镜头的"选择此镜头"按钮
- **行为**：用选中的备选镜头替换推荐镜头
- **反馈**：推荐镜头更新为新选择的镜头

#### 操作 3：禁用镜头
- **触发**：点击"禁用"按钮
- **行为**：标记此镜头为"不要再用"，在后续生成中不会再选用
- **反馈**：镜头被标记为"已禁用"；系统自动选择下一个备选镜头或使用替代呈现

#### 操作 4：标记"纪念性必保留"
- **触发**：点击"纪念性必保留"按钮
- **行为**：标记此镜头为"纪念性必保留"，系统会优先保留在最终成片中
- **反馈**：镜头被标记为"纪念性必保留"

#### 操作 5：接受替代呈现
- **触发**：点击"接受"按钮（在替代呈现区域）
- **行为**：接受照片+旁白过渡的替代方案
- **反馈**：替代呈现被标记为"已接受"

#### 操作 6：替换照片
- **触发**：点击"替换照片"按钮
- **行为**：打开照片选择器，允许用户选择其他照片
- **反馈**：照片列表更新

### 6.5 状态反馈

| 状态 | 显示 | 说明 |
|------|------|------|
| 正常 | 镜头卡片正常显示 | 无问题 |
| 低置信 | 黄色警告标记 | 置信度 < 0.5，建议用户审阅 |
| 已接受 | 绿色勾选 | 用户已接受此镜头 |
| 已禁用 | 灰色标记 | 用户已禁用此镜头 |
| 纪念性必保留 | 红色星标 | 用户标记为纪念性必保留 |
| 替代呈现 | 蓝色标记 | 使用照片+旁白过渡替代 |

### 5.6 确认与返回

- **返回故事骨架确认**：点击"返回故事骨架确认"按钮，返回故事骨架确认页面重新调整
- **返回**：点击"返回"按钮，返回上一步

### 6.4 用户操作

- **触发**：点击"选择操作"下拉菜单
- **行为**：显示三个入口选项
- **反馈**：下拉菜单展开

#### 操作 2：仅重写旁白
- **触发**：选择"仅重写旁白"
- **行为**：
  - 打开旁白编辑界面
  - 用户可以修改旁白文案
  - 系统重新生成 TTS 音频
  - 系统重新混音
  - 系统重新导出
- **反馈**：新版本生成完成，添加到版本历史

#### 操作 3：仅更换 BGM 并重混音
- **触发**：选择"仅更换 BGM 并重混音"
- **行为**：
  - 打开 BGM 选择界面
  - 用户选择新的 BGM
  - 系统重新混音
  - 系统重新导出
- **反馈**：新版本生成完成，添加到版本历史

#### 操作 4：压缩到更短时长
- **触发**：选择"压缩到更短时长"
- **行为**：
  - 打开时长设置界面
  - 用户指定目标时长
  - 系统基于既有故事骨架和候选镜头重排节奏
  - 系统重新导出
- **反馈**：新版本生成完成，添加到版本历史

#### 操作 5：预览版本
- **触发**：点击版本卡片的"预览"按钮
- **行为**：在播放器中预览该版本
- **反馈**：播放器加载并播放该版本

#### 操作 6：设为当前版本
- **触发**：点击版本卡片的"设为当前版本"按钮
- **行为**：将该版本设为当前版本
- **反馈**：当前版本标识更新；播放器显示新版本

#### 操作 7：删除版本
- **触发**：点击版本卡片的"删除"按钮
- **行为**：删除该版本（不能删除当前版本）
- **反馈**：版本列表更新

#### 操作 8：导出当前版本
- **触发**：点击"导出当前版本"按钮
- **行为**：导出当前版本为 MP4、字幕、配音文件等
- **反馈**：导出完成，显示导出路径

## 7. 错误提示与诊断信息

### 7.1 错误提示位置

所有错误提示都显示在**对应段落旁边**，采用以下样式：

```
┌─ 段落 1 ─────────────────────────────────────────┐
│ 摘要：第一天的旅程开始...                        │
│ 时间范围：2026-04-01 09:00 - 14:30              │
│                                                   │
│ ⚠️ 低置信匹配                                    │
│ 这个段落的匹配置信度 0.45，因为缺少 GPS 数据。  │
│ 建议：检查素材的位置信息，或手动调整候选镜头。  │
│                                                   │
│ [操作] ▼                                          │
└───────────────────────────────────────────────────┘
```

### 7.2 简化版诊断信息字段

用户界面显示的诊断信息应包含以下字段（简化版）：

| 字段 | 说明 |
|------|------|
| 问题类型 | 低置信匹配、素材不足、元数据缺失等 |
| 置信度数值 | 具体的置信度分数（如 0.45） |
| 根本原因 | 为什么出现这个问题（如"缺少 GPS 数据"） |
| 用户提示 | 易读的文案，告诉用户发生了什么 |
| 建议动作 | 建议用户采取的修复方案 |

**注**：后台系统仍然记录完整的诊断信息（包括 issue_type、confidence_score、root_cause、suggested_action 等），但用户界面只显示上述简化版字段。

### 7.3 诊断信息示例

```
⚠️ 低置信匹配

```

```
⚠️ 素材不足

⚠️ 元数据缺失
部分素材缺少 GPS 数据，系统已切换为语义优先对齐。
建议：对齐结果可能不如预期，建议在高光确认时仔细审阅。
```

## 9. 交互流程总结

```
项目创建
    ↓
素材导入
    ↓
游记输入
    ↓
BGM 选择
    ↓
TTS 音色选择
    ↓
故事骨架确认
    ├─ 用户审阅、修改、确认
    └─ 冻结故事骨架版本
    ↓
素材分析与对齐（自动）
    ↓
高光确认
    ├─ 用户审阅、修改、确认
    └─ 冻结高光选择版本
    ↓
自动成片生成
    ↓
第一版成片预览
    ├─ 用户评分
    └─ 用户可进行局部再生成
    ↓
局部再生成（可选）
    ├─ 仅重写旁白
    ├─ 仅更换 BGM 并重混音
    ├─ 压缩到更短时长
    └─ 生成新版本
    ↓
版本切换与导出
    ├─ 用户在版本间切换
    └─ 用户导出最终成片
```
