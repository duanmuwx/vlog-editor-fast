# 常见问题 (FAQ)

## 功能相关

### Q: 系统支持哪些视频格式？
A: 支持所有 FFmpeg 支持的格式，包括：
- MP4 (H.264, H.265)
- MOV (Apple ProRes)
- AVI (MPEG-4)
- MKV (VP9, H.264)
- WebM (VP8, VP9)

### Q: 支持哪些图片格式？
A: 支持所有常见图片格式：
- JPG/JPEG
- PNG
- GIF
- BMP
- TIFF
- WebP

### Q: 游记需要多长？
A: 推荐 3000-5000 字。系统支持：
- 最短：150 字（使用简化模式）
- 推荐：3000-5000 字
- 最长：无限制（但会影响处理时间）

### Q: 需要多少媒体文件？
A: 推荐配置：
- 视频：20-50 个
- 图片：300-600 张
- 总时长：60-120 分钟

最少配置：
- 视频：5 个 或 图片：100 张

### Q: 支持多语言吗？
A: 是的，系统支持：
- 游记：中文、英文、日文、韩文等
- 旁白：支持多种 TTS 音色
- 字幕：自动生成对应语言

## 性能相关

### Q: 处理一个标准项目需要多长时间？
A: 标准项目（30 个视频、400 张图片、4000 字游记）通常需要：
- 快速模式：60-90 分钟
- 标准模式：90-120 分钟
- 高质量模式：120-180 分钟

### Q: 如何加快处理速度？
A: 几种方法：
1. 使用轻量模式（降低质量）
2. 减少媒体文件数量
3. 使用更快的硬件（SSD、多核 CPU）
4. 启用硬件加速（GPU）

### Q: 需要多少磁盘空间？
A: 标准项目需要：
- 输入文件：~5GB
- 中间产物：~3GB
- 最终成片：~500MB
- 总计：~8-10GB

### Q: 需要多少内存？
A: 推荐配置：
- 最少：4GB
- 推荐：8GB
- 最优：16GB+

### Q: 可以离线使用吗？
A: 大部分功能可以离线使用，除了：
- 故事解析（需要 LLM API）
- 可选的云端功能

## 功能使用

### Q: 如何调整故事骨架？
A: 在故事骨架确认阶段，可以：
1. **合并段落** - 将相关段落合并
2. **删除段落** - 移除不重要的段落
3. **重新排序** - 调整段落顺序
4. **编辑摘要** - 修改段落描述

### Q: 如何替换高光镜头？
A: 在高光确认阶段：
1. 查看每个段落的候选镜头
2. 点击替换按钮
3. 选择其他候选镜头
4. 或手动选择媒体文件

### Q: 什么是替代呈现单元？
A: 当没有合适的视频时，系统使用：
- **照片 + 旁白** - 静止照片配合旁白
- **位置卡片** - 显示地点信息
- **文字卡片** - 显示文字描述

### Q: 如何进行局部再生成？
A: 完成初始成片后，可以：
1. **仅重写旁白** - 修改旁白文案
2. **仅更换 BGM** - 选择新的背景音乐
3. **压缩时长** - 缩短视频时长

### Q: 如何处理低置信度匹配？
A: 系统会：
1. 提示用户确认
2. 显示多个候选镜头
3. 允许手动选择
4. 或自动使用替代呈现单元

## 版本管理

### Q: 如何保存不同版本？
A: 系统自动保存所有版本：
- 每个阶段都会创建版本
- 可以随时切换版本
- 可以比较版本差异

### Q: 如何恢复到之前的版本？
A: 使用版本切换功能：
```bash
POST /api/projects/{project_id}/versions/{artifact_type}/{version_id}/switch
```

### Q: 如何删除旧版本？
A: 系统会自动管理版本：
- 保留最近 10 个版本
- 旧版本自动清理
- 可以手动标记为删除

## 导出相关

### Q: 支持哪些导出格式？
A: 目前支持：
- MP4 (H.264, H.265)
- 字幕文件 (SRT, VTT)
- 配音文件 (MP3, WAV)
- 项目清单 (JSON)

### Q: 如何调整导出质量？
A: 导出时可以选择：
- **低质量** - 快速导出，文件小
- **中等质量** - 平衡速度和质量
- **高质量** - 最佳质量，文件大

### Q: 导出失败怎么办？
A: 系统会自动重试：
1. 第一次失败 - 使用相同设置重试
2. 第二次失败 - 降低质量重试
3. 第三次失败 - 使用替代编码器重试

### Q: 可以导出为其他格式吗？
A: 目前不支持，但可以：
1. 导出为 MP4
2. 使用 FFmpeg 转换为其他格式

```bash
ffmpeg -i output.mp4 -c:v libvpx -c:a libopus output.webm
```

## 故障排除

### Q: 为什么项目创建失败？
A: 常见原因：
1. 游记太短（< 150 字）
2. 媒体文件不足
3. 文件格式不支持
4. 文件路径不正确

### Q: 为什么故事解析失败？
A: 常见原因：
1. API 密钥不正确
2. 网络连接问题
3. API 配额已用尽
4. 游记格式不正确

### Q: 为什么对齐置信度低？
A: 常见原因：
1. 媒体与故事不匹配
2. 元数据缺失
3. 媒体质量差
4. 游记描述不清楚

### Q: 为什么导出失败？
A: 常见原因：
1. 磁盘空间不足
2. FFmpeg 未安装
3. 编码器不可用
4. 文件权限问题

## 技术相关

### Q: 系统使用什么技术栈？
A: 
- 后端：Python 3.10+, FastAPI
- 数据库：SQLite
- 媒体处理：FFmpeg, OpenCV
- 音频处理：librosa
- 语音识别：Whisper
- LLM：Kimi API

### Q: 支持 GPU 加速吗？
A: 是的，支持：
- NVIDIA CUDA（H.264/H.265 编码）
- Apple Video Toolbox（Mac）
- Intel Quick Sync（Intel CPU）

### Q: 可以在云端运行吗？
A: 可以，支持：
- Docker 容器化
- Kubernetes 部署
- AWS/Azure/GCP 云平台

### Q: 支持 API 集成吗？
A: 是的，提供完整的 REST API：
- 项目管理
- 故事处理
- 媒体分析
- 版本管理
- 诊断报告
